# 格式一：简历呈现版 (The Resume)

- **定位**：高密度信息压缩包。
- **目标读者**：HR (关键词匹配)、技术面试官 (快速筛选)、ATS 系统 (解析率)。
- **前置要求**：请先阅读并签署 [通用基线 (The Baseline Protocol)](./project-experience-baseline.md)。本规范建立在基线的 **真实性公理** 与 **语言净化过滤器** 之上。
- **AI 生成指令**：作为“精炼器”，将庞大的技术细节压缩为 3-4 行高价值“子弹点 (Bullet Points)”。

## 1. 信息压缩算法 (Compression Algorithm)

在生成简历内容时，请遵循以下逻辑流：

### 第一步：信号提取 (Signal Extraction)

从白皮书或代码中提取以下高价值信号（对应基线 **STAR-Plus** 的各维度）：

1.  **量级 (Scale)**：并发数、数据量、代码行数、模块数 (Situation)。
2.  **核心动作 (Key Actions)**：重构、优化、搭建、设计 (Action)。
3.  **具体技术 (Tech Keywords)**：React, Turborepo, Docker (Action)。
4.  **量化结果 (Metrics)**：ms, %, MB, QPS (Result)。

### 第二步：句式重组 (Sentence Construction)

使用 **STAR-L (Lite)** 结构进行组装。这是基线 **STAR-Plus** 的“压缩版”：

- `[动作谓语] + [技术对象] + [具体手段] + [量化结果]`

> **原始素材**：
> "这个项目里我负责优化了列表页，以前很卡，加载要 3 秒，我用了虚拟滚动，还把图片懒加载了，现在只要 1 秒多了。"
>
> **处理过程**：
>
> 1. 提取：优化列表页 (Action), 虚拟滚动/懒加载 (Tech), 3s->1s (Result).
> 2. 重组：**重构**商品长列表，引入 **vue-virtual-scroller** 与 **IntersectionObserver** 实现按需渲染，将首屏 FCP 从 **3.2s 降至 1.1s**。

### 第三步：密度检查 (Density Check)

请对照 [通用基线 - 语言净化过滤器](./project-experience-baseline.md#2-语言净化过滤器-nlp-filter) 进行自检，并额外确认：

- [ ] **无废话**：是否清除了“黑名单”中的虚词？
- [ ] **无孤儿技术栈**：技术关键词是否紧跟解决了什么问题？
- [ ] **数字显性化**：数字需加粗或前置。

## 2. 排版渲染规范 (Rendering Rules)

### 头部元数据 (Header)

- **格式**：`[项目名称] | [角色] | [时间]`
- **简介**：用 **1 句话** 概括业务与技术栈。
  - _Template_: `[业务类型]平台，服务于[用户群体]，日均[PV/UV/单量]。技术栈：[核心技术1] + [核心技术2] + ...`

### 核心职责 (Responsibilities)

- **数量限制**：严格控制在 **3-4 条**。
- **排序逻辑**：
  1.  **亮点 (Highlight)**：最难的技术攻坚 / 最大的性能提升。
  2.  **架构/工程 (Architecture)**：基建、规范、通用组件 (Legacy)。
  3.  **业务复杂度 (Complexity)**：解决的最复杂的业务逻辑。
  4.  _(可选) 质量/稳定性 (Quality)_：监控、测试、Bug 率。
