# 职业发展规划

（Front-end Developer & 海外发展方向）

## 四、职业发展目标（时间轴）

### 4.1 短期目标（未来 1–2 年）

**总体目标：**
在国内提升前端专业度 + 英语能力，同时系统研究海外深造/工作路径，为后续「读硕 or 直接出国」打基础。

**具体目标：**

1. 技术成长

   - 夯实三大块：

     - 前端基础（JS / TS、浏览器原理、性能优化）
     - 主力框架生态（如 React + 周边：状态管理、路由、SSR 等）
     - 工程化能力（构建、部署、监控、日志、调试）

   - 每年至少主导或核心参与【2–3】个功能模块或小项目，并形成**可公开展示的项目说明（中英文）**。

2. 英语能力

   - 英文阅读保持现有水平，全面使用英文文档 / 官方资料
   - 提升听说：每周至少【X】小时听力 / 口语训练（如英语角、在线 1v1、跟读技术大会视频）
   - 一年内完成一版**英文简历 + LinkedIn 个人主页 + GitHub README（英文版）**。

3. 海外方案调研

   - 选定 2–3 个潜在目标地区，例如：【日本、新加坡、德国、某英语国家】
   - 对每个地区收集以下信息：

     - 典型 CS / IT / Software Engineering 硕士项目（学制、学费、语言要求）
     - 工作签证政策与求职窗口（例如毕业后可以留多久找工作）
     - 程序员就业市场大致情况

   - 形成一份个人**《海外发展方案对比表》**（包含：成本、难度、回报预期）。

### 4.2 中期目标（3–5 年）

根据 1–2 年的探索结果，选择以下路径之一为主线：

1. **路径 A：读海外硕士 + 本地找工作**

   - 申请 1–2 所性价比合理的硕士项目
   - 控制总预算在【X】万以内（含生活费）
   - 在硕士期间积极寻找实习和项目机会，为毕业后留下来工作铺路

2. **路径 B：在国内积累到 3–5 年经验后冲海外工作 / 远程**

   - 在国内定位为高级前端 / 工程化专家
   - 有较强的作品集、技术博客、开源贡献
   - 以远程岗位、跨国公司中国分部、内部转岗等形式进入国际化团队

3. **路径 C：读国内/性价比高的研究生 + 国际化公司**

   - 利用研究生阶段参与更多科研/大厂实习
   - 优先进入有全球团队的公司，为后续海外调动创造机会

> 这三条路径可以并行准备，到第 3 年左右再根据经济状况、能力提升和个人意愿进行选择。

### 4.3 长期目标（5 年以上）

- 在一个高度国际化的团队中长期工作（地点可以是国外，也可以是中国但团队全球化）
- 在某个技术细分方向建立明确标签（如“前端工程化”“高性能数据可视化”“文档/报表系统专家”等）
- 保持有选择权：既能在国内找到满意工作，也能在有机会时切换至海外岗位

## 五、能力建设与学习规划

### 5.1 技术能力

- 每年重点攻克【1–2】个深度技术方向，例如：

  - 前端工程化体系（Monorepo、CI/CD、模块化、打包优化）
  - 前端性能优化与大规模表格/报表渲染
  - 富文本 / PDF 报表编辑器等偏复杂组件领域

- 制定年度学习计划：

  - 精读【X】本专业书 / 文档
  - 完成【X】个技术分享（公司内 / 线上）
  - 输出【X】篇技术文章（中英文均可，发布到个人博客 / 掘金 / GitHub Pages）

### 5.2 语言与软技能

- 英语：

  - 目标：能用英语参加技术面试，并在工作场景中完成日常沟通与书面表达
  - 方式：看英文技术大会、读 RFC/官方 docs、与外教或外国友人练口语

- 软技能：

  - 需求沟通、需求拆分与排期估算
  - 与后端 / 产品 / 设计的跨角色协作
  - 写清晰的技术文档与方案

## 七、风险评估与应对策略

### 7.1 主要风险

1. 读硕士投入大，但未必能留在当地工作
2. 海外求职时间不确定，可能有一段时间收入中断
3. 技术路线选择失误（例如选择了在海外市场需求较弱的方向）
4. 对国内岗位与竞争情况判断不足，可能回来后产生落差

### 7.2 应对策略

- 严格控制预算，提前设定**止损线**
- 在任何时候，都保持国内就业竞争力（技术栈不过时、有能拿得出手的项目）
- 在做重大决策前，多向有实际经验的前辈请教（如有海外留学/工作经历的学长学姐）
- 定期（每 6–12 个月）回顾：

  - 当前路线是否仍匹配自己的价值观和生活状态
  - 是否需要调整目标国家 / 路线
