# gel-api 架构说明

## 目录职责
本模块是项目的 API 请求层，封装了 `axios`，提供了统一的 API 调用函数、请求/响应拦截器、以及环境判断等功能。

## 目录结构
```
├── src/                      # 源代码
│   ├── core/                 # 核心 axios 实例和拦截器
│   ├── services/             # 按业务模块划分的 API 服务
│   ├── env/                  # 环境判断相关函数
│   └── index.ts              # 主入口，统一导出
├── dist/                     # Vite 打包后的产物
├── doc/                      # 相关文档
├── vite.config.ts            # Vite 构建配置文件
└── package.json              # 项目依赖和脚本配置
```

## 文件职责说明
- **src/core/**: 创建和配置 `axios` 实例，设置通用的请求头、超时和拦截器。
- **src/services/**: 存放具体的 API 请求函数，通常按业务领域划分。
- **src/env/**: 提供用于判断当前环境（如开发、生产、客户端）的工具函数。
- **src/index.ts**: 统一导出所有公共的 API 服务和工具函数。

## 模块依赖
- **gel-types**: 依赖共享的 TypeScript 类型来定义 API 的请求和响应体。
- **axios**: 作为 HTTP 请求的基础库。
- 被所有需要进行网络请求的应用和包所依赖。
