# 前端简历项目经历规范

## 核心原则

### STAR 结构

- **Situation**：项目背景（用户量、业务场景）
- **Task**：你的职责（负责的模块）
- **Action**：技术方案和实现
- **Result**：量化成果（性能 + 业务指标）

### 量化指标

- 性能：首屏时间、FCP、LCP、包体积
- 业务：转化率、留存率、PV/UV
- 效率：开发效率、代码复用率、构建时间

### 技术栈

- 聚焦主流框架生态（React/Vue + 工具链）
- 体现工程化能力（构建、状态管理、测试）
- 展示性能优化（SSR、懒加载、缓存）

## 标准格式

```
【项目名称】 | 时间跨度
项目简介：项目类型 + 用户规模 + 核心价值
技术栈：框架 + 状态管理 + 构建工具 + UI 库
职责与成果：
- 开发 XX 模块，实现 XX 功能，首屏时间 X ms，转化率提升 X%
- 优化 XX 性能，通过 XX 方案将 XX 从 X 降至 Y
- 搭建 XX 体系（组件库/监控），提升效率 X%
```

## 示例对比

### ❌ 错误

```
参与电商平台开发，使用 Vue、React 等框架，负责一些页面开发，
学习了很多新技术，提升了自己的能力。
```

问题：动词模糊、技术堆砌、无量化、关注自我成长

### ✅ 正确

```
【电商平台前端重构】 | 2023.06 - 2023.12
项目简介：100 万+ DAU 的 B2C 电商平台架构升级
技术栈：Vue 3 + TypeScript + Vite + Pinia + Element Plus
职责与成果：
- 主导商品详情页重构，虚拟滚动优化长列表，首屏从 3.2s 降至 1.1s
- 实现图片懒加载和 WebP 适配，资源体积减少 45%，LCP 提升 40%
- 封装 15+ 业务组件形成组件库，新页面开发效率提升 50%
```

## 动词清单

**推荐**：开发、实现、构建、搭建、封装、优化、重构、主导、推动
**避免**：负责、参与、了解、学习、接触

## 经验层级侧重

### 1-2 年

- 独立完成的功能模块
- 主流框架熟练使用
- 代码质量和规范意识

### 3-5 年

- 性能优化和技术攻坚
- 组件库/工具库沉淀
- 工程化体系搭建

### 5 年以上

- 技术选型和架构决策
- 团队规范和基础设施
- 跨团队协作和影响力

## 检查清单

- [ ] 使用具体数字量化成果？
- [ ] 说明技术方案和实现？
- [ ] 动词具体且主动？
- [ ] 技术栈聚焦且匹配岗位？
- [ ] 每个项目 3-5 条，每条 1-2 行？
- [ ] 项目按时间倒序排列？
- [ ] 选择最能体现能力的 3-4 个项目？
