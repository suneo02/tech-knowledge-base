# Interview Project Dossier

## Purpose

Convert a project into interview-prep assets: a full dossier (whitepaper) and, when requested, an oral script. Keep outputs evidence-backed, text-only, and interview-ready.

## Modes

- Dossier mode (whitepaper): full-coverage project dossier with evidence anchors.
- Script mode: oral hook + STAR-Plus deep dive + defensive Q&A.

## Quick Start

- Choose mode.
- Locate the matching `*-facts.md` file under `private/carrier/resume/projects/` and read it first.
- If no facts file exists, do not draft the dossier yet; route to `resume-project-facts`.
- Always apply baseline rules: read `references/baseline.md`.
- Dossier mode: read `references/dossier-guide.md` and `references/dossier-template.md`.
- Script mode: read `references/script.md`.
- If the user only needs resume bullets, route to `interview-project-resume`.

## Core Constraints

- Output language: Chinese.
- Text-only Markdown; no code blocks, diagrams, or tables.
- First-person narrative (我...).
- Evidence anchors required for key claims.
- The dossier may organize and compress facts, but must not exceed the boundaries established in `*-facts.md`.
- If dossier materials conflict with `*-facts.md`, treat the facts file as authoritative.

## Workflow (High Level)

1. Decide mode and scope.
2. Read the matching `*-facts.md` file and treat it as the primary truth source.
3. Gather supporting inputs per mode (see references).
4. Draft output using the mode template/format.
5. Validate against baseline, facts boundaries, and mode checklist.

## Routing Rule

- `facts` answers what is true.
- This skill answers how to narrate those truths for interview use.
- If the user is still clarifying real scope, ownership, or safe claims, do not continue here; route to `resume-project-facts`.

## References

- `references/baseline.md`: truthfulness, language filter, leveling, STAR-Plus.
- `references/dossier-guide.md`: dossier inputs, workflow, style guidance, validation, questions.
- `references/dossier-template.md`: dossier template.
- `references/script.md`: script inputs, workflow, narrative, defensive Q&A, output format.
