# 面试题文档规范

## 写作原则

- 70% 以上使用自然语言解释思路，代码仅作辅助。
- 先给结论与核心思路，再补充细节与权衡。
- 关注场景与决策理由，避免堆砌概念。
- 单题建议控制在 100-300 行内，场景合集可在 300-1000 行内并保持清晰分节。
- 超过约 1000 行且包含多个独立子主题时再拆分。

## 推荐结构

~~~markdown
# 问题标题
> 一句话概览

## 核心思路
- 关键结论
- 决策依据

## 方案对比
- 方案 A：适用场景/优缺点
- 方案 B：适用场景/优缺点

## 技术要点
- 要点 1
- 要点 2

## 精简代码（可选）
```lang
// 核心逻辑
```

## 常见追问
- 追问 1
- 追问 2
~~~

## 常见追问示例

- 为什么不用 X？
- 在高并发/弱网场景下怎么处理？
- 如何验证/排查问题？
