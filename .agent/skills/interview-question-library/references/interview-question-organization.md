# 面试题库组织与维护

## 目录与命名

- 保持领域结构：以 `library/` 下的领域目录为主，不随意跨域移动。
- 避免深层目录：单题不要新建目录；需要拆分时再建二级目录。
- 拆分规则：文档超过约 1000 行且包含多个独立子主题时才拆分。
- 入口文件：目录必须有 `README.md` 或 `index.md` 作为索引。
- 命名统一使用 kebab-case。

## 索引与链接

- 新增或拆分内容后，更新最近的 `README.md` 索引。
- 统一使用相对路径进行站内链接。
- 同域优先链接，跨域仅在必要时添加。

## 优先级与频率标注

- 仅在用户明确要求或已有惯例时引入标注。
- 标注方式跟随现有模式（例如 `p0/p1/p2` 或 `high/mid/low`）。
- 避免创建新的全局标签体系，除非用户批准。

## 维护检查

- 单一 H1，标题层级正确。
- 代码块标注语言且只保留核心逻辑。
- 关键结论突出，避免冗余或冲突内容。
- 资源路径与相对链接可用。
