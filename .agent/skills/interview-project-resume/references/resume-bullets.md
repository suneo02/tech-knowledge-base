# 格式一：面试简历版 (Interview Resume Bullets)

- **定位**：面试用项目经历的高密度信息压缩包，1 分钟内读懂价值点。
- **目标读者**：HR（关键词匹配）、技术面试官（快速筛选）、ATS 系统（解析率）。
- **输入来源**：白皮书/源码/数据面板/复盘笔记（必须可回溯）。
- **前置要求**：遵守 [面试基线](./baseline.md) 的 **真实性公理** 与 **语言净化过滤器**。
- **输出形态**：每个项目 **1 行简介 + 3-4 条 Bullet Points**。
- **AI 生成指令**：作为“精炼器”，将庞大的技术细节压缩为 3-4 行高价值 Bullet Points。

## 1. 信息压缩算法 (Compression Algorithm)

### 第一步：信号提取 (Signal Extraction)

从白皮书或代码中提取以下高价值信号（对应基线 **STAR-Plus** 的各维度）：

1. **量级 (Scale)**：并发数、数据量、代码行数、模块数（Situation）。
2. **核心动作 (Key Actions)**：重构、优化、搭建、设计（Action）。
3. **具体技术 (Tech Keywords)**：React、Turborepo、Docker（Action）。
4. **量化结果 (Metrics)**：ms、%、MB、QPS（Result）。
5. **资产沉淀 (Legacy, 可选)**：组件库、规范、文档（Legacy）。

### 第二步：句式重组 (Sentence Construction)

使用 **STAR-L (Lite)** 结构进行组装（基线 **STAR-Plus** 的压缩版）：

- `[动作谓语] + [技术对象] + [具体手段] + [量化结果]`

> **原始素材**：
> “这个项目里我负责优化了列表页，以前很卡，加载要 3 秒，我用了虚拟滚动，还把图片懒加载了，现在只要 1 秒多了。”
>
> **处理过程**：
> 1. 提取：优化列表页（Action），虚拟滚动/懒加载（Tech），3s -> 1s（Result）。
> 2. 重组：**重构**商品长列表，引入 **vue-virtual-scroller** 与 **IntersectionObserver** 实现按需渲染，将首屏 FCP 从 **3.2s 降至 1.1s**。

### 第三步：密度检查 (Density Check)

对照 [面试基线 - 语言净化过滤器](./baseline.md#2-语言净化过滤器-nlp-filter) 自检，并额外确认：

- [ ] **无废话**：是否清除“黑名单”中的虚词？
- [ ] **无孤儿技术栈**：技术关键词是否紧跟解决了什么问题？
- [ ] **数字显性化**：数字需加粗或前置。
- [ ] **可回溯**：每个技术点能定位到代码或文档。

## 2. 排版渲染规范 (Rendering Rules)

### 头部元数据 (Header)

- **格式**：`[项目名称] | [角色] | [时间]`
- **简介**：用 **1 句话** 概括业务与技术栈。
  - _Template_: `[业务类型]平台，服务于[用户群体]，日均[PV/UV/单量]。技术栈：[核心技术1] + [核心技术2] + ...`

### 核心职责 (Responsibilities)

- **数量限制**：严格控制在 **3-4 条**。
- **排序逻辑**：
  1. **亮点 (Highlight)**：最难的技术攻坚 / 最大的性能提升。
  2. **架构/工程 (Architecture)**：基建、规范、通用组件（Legacy）。
  3. **业务复杂度 (Complexity)**：解决的最复杂业务逻辑。
  4. _(可选) 质量/稳定性 (Quality)_：监控、测试、Bug 率。
- **句式要求**：动作词开头，紧跟技术手段与量化结果；无数据时用“用户反馈/成本下降/事故减少”替代。
- **ATS 友好**：避免 emoji 与花体符号，使用标准标点。

## 3. 输出模板 (Copy & Fill)

```markdown
[项目名称] | [角色] | [时间]
[业务类型]平台，服务于[用户群体]，日均[PV/UV/单量]。技术栈：[核心技术1] + [核心技术2] + ...
- [动词] [对象/模块]，通过 [技术手段] 实现 [结果/指标]
- [动词] [对象/模块]，引入 [技术/机制] 将 [指标] 从 [A] 提升/降低至 [B]
- [动词] [对象/模块]，构建 [资产/规范]，支撑 [业务场景]
- (可选) [动词] [质量/稳定性项]，将 [事故/耗时/成本] 降低 [比例]
```
