# 前端设计文档编写规范

## 快速准则
- 仅聚焦功能流与可见状态，删去与实现、性能等非功能描述。
- 页面按用户任务拆分，标出主次信息与控件反馈。
- 每个任务至少配一张结构或流程图，明确加载/空/错误/提交四类状态。

## 推荐结构
1. **概览**：功能边界、目标人群、3 条以内的关键场景。
2. **信息架构**：Mermaid 图描述入口、跳转与主要页面。
3. **页面蓝图**：用表格列区域、展示数据、可交互动作、可见条件。
4. **任务流程**：流程/状态图覆盖提交链路与异常回退。
5. **数据与状态**：字段来源、过滤/排序规则、缓存或刷新策略。
6. **组件复用**：列出复用组件/Hook、输入输出契约、错误捕获点。

## 写作提示
- 用 Mermaid、表格描述结构，正文只解释决策与限制。
- 交互状态差异（禁用、二次确认、撤销）独立说明触发与反馈。
- 引用现有组件/Hook/样式时使用相对路径添加 `@see`。
- 文档修改与代码改动同步提交，并在 PR 描述中互相引用。

## 检查清单
- [ ] 每个页面或任务至少有一张图/表。
- [ ] 加载、空、错误、成功状态及跳转路径明确。
- [ ] 数据字段、筛选、排序、刷新策略写清。
- [ ] 组件边界与复用说明含错误处理点。
- [ ] 全文控制在 1 页，并包含代码 `@see`。
