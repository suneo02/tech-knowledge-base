# Requirement Doc Rules

## Quick Standards

- **Language**: All content in Chinese.
- **Perspective**: User-centric. "What to do", NOT "How to build".
- **Scope**: Functional only. No technical details.
- **Format**: Diagrams/Tables > Text. Max 1 page.

## Structure

1. **Overview (概览)**: 描述目标用户与核心场景。
2. **Modules (模块)**: 用示意图或列表说明区域划分与主要元素。
3. **Flows (用户流程)**: 用流程图或编号步骤描述关键任务。
4. **Data & Rules (数据与规则)**: 列出展示字段、输入校验、状态切换。
5. **Feedback (反馈)**: 说明成功、失败、加载、空态的用户感知。

## Checklist

- [ ] Clear User & Goal.
- [ ] Flows visualized.
- [ ] All states (Success/Fail/Loading) covered.
- [ ] Linked to Design/Code via `@see`.
