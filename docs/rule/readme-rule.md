# README 文档编写规范

## 快速准则
- 读者需在 3 分钟内掌握目录“是什么、有什么”。
- 内容聚焦用途、结构、关联文档；不写实现细节、部署流程或其他非功能信息。
- 文档长度 ≤100 行（复杂模块上限 150 行），所有引用使用相对路径。

## 推荐结构
1. **一句话定位**：说明该目录负责的功能或业务。
2. **目录树**：展示完整结构，并在每行括注职责。
3. **关键文件说明**：列出 3-5 个核心文件及作用。
4. **依赖示意**：用箭头或短列表交代依赖关系/上下游。
5. **相关文档**：链接到设计、需求、接口或问题记录。

## 写作提示
- 应用目录强调业务模块与依赖的公共包。
- 组件/库目录列出输出能力及被哪些应用使用。
- 功能模块突出与其他模块的边界和入口路径。
- 每段内容都应回答“这个目录的功能是什么”，其他非功能性说明移除或改为链接。
- 关键模块必须引用对应设计/方案，并在代码中 `@see` 回指。

## 检查清单
- [ ] 目录树完整且注释清晰。
- [ ] 未包含开发规范、API、部署、性能等与功能无关的冗余信息。
- [ ] 依赖关系描述一目了然。
- [ ] 文档长度符合要求。
- [ ] 相关文档链接使用相对路径。
