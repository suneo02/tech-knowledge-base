# 项目整体架构

## 1. 项目目录结构

```
.
├── apps/                  # 应用程序目录
│   └── report-config/     # 报表配置应用
├── packages/              # 可共享的包目录
│   ├── ai-chat/           # AI 聊天相关功能
│   ├── cde/               # CDE 相关功能
│   ├── detail-page-config/ # 详情页面配置
│   ├── gel-api/           # GEL 项目 API 后续API统一使用该文件夹维护
│   ├── gel-ui/            # GEL 项目 UI 组件 后续涉及到通用组件统一使用该文件夹维护
│   ├── gel-util/          # GEL 项目工具库
│   ├── indicator/         # 指标相关功能
├── scripts/               # 项目脚本
├── .cursor/               # Cursor AI 相关配置
├── node_modules/          # Node.js 模块
├── .npmrc                 # npm 配置文件
├── .prettierrc            # Prettier 代码格式化配置
├── api-proxy-server.js    # API 代理服务器脚本
├── package.json           # 项目元数据和依赖管理
├── pnpm-lock.yaml         # pnpm 锁文件
├── pnpm-workspace.yaml    # pnpm 工作区配置
├── README.md              # 项目说明文档
├── TURBOREPO.md           # Turborepo 相关说明
├── turbo.json             # Turborepo 配置文件
```

## 2. `packages` 目录下的项目

`packages` 目录包含了多个可共享的子项目，具体如下：

- `ai-chat`: AI 聊天相关功能
- `cde`: CDE 相关功能
- `detail-page-config`: 详情页面配置
- `gel-api`: GEL 项目 API
- `gel-ui`: GEL 项目 UI 组件
- `gel-util`: GEL 项目工具库
- `indicator`: 指标相关功能
- `types`: 类型定义

## 3. 主要插件包 (devDependencies)

项目使用了一些开发依赖包来辅助开发、构建和测试等任务，主要包括：

- `@types/lodash`: Lodash 的 TypeScript 类型定义。
- `@types/node`: Node.js 的 TypeScript 类型定义。
- `concurrently`: 同时运行多个命令的工具。
- `cors`: Express 中间件，用于启用 CORS (跨源资源共享)。
- `dotenv`: 用于从 `.env` 文件加载环境变量。
- `express`: Node.js 的 Web 应用框架。
- `http-proxy-middleware`: Node.js 的 HTTP 代理中间件。
- `turbo`: 高性能的 Monorepo 构建系统。
- `wait-on`: 用于等待文件、端口、HTTP 资源等可用。
