## 核心需求

用户在大纲会话中上传文件（`files`）和引用文件（`refFiles`）后，需要：

- 自动轮询文件解析状态
- 实时更新文件解析进度
- 解析完成后停止轮询
- 支持多文件同时处理

## 技术要点

- 复用现有 `useFileStatusPolling` Hook
- 从消息中提取待解析文件
- 更新消息中的文件状态
- 集成到大纲会话 Context
