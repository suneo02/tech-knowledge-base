# Testing Rules

## Structure
- `__tests__/` (Vitest)
- `stories/` (Storybook)
- `__mocks__/` (Shared mocks)

## Requirements
- **Language**: Test descriptions (`describe`/`it`) in Chinese.
- **Mandatory**: New features (functions/components) must have tests.
- **Naming**: `*.test.ts(x)`, `*.stories.tsx`.

## Checklist
- [ ] Directory structure followed.
- [ ] Tests written and passing for new code.
