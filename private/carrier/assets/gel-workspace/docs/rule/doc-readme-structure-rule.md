# README Structure Rules

## Quick Standards

- **Language**: All content in Chinese.
- **Goal**: 3-minute read. Explain "What is this" and "What's inside".
- **Content**: Functional scope only. No implementation details.
- **Links**: Use relative paths to other docs.

## Structure

1. **One-liner (一句话定位)**: 说明该目录负责的功能或业务。
2. **Tree (目录树)**: 展示完整结构，并在每行括注职责（中文）。
3. **Key Files (关键文件)**: 列出 3-5 个核心文件及作用。
4. **Dependencies (依赖)**: 交代依赖关系/上下游。
5. **Docs (相关文档)**: 链接到设计、需求、接口或问题记录。

## Checklist

- [ ] Tree is complete & annotated.
- [ ] Dependencies are clear.
- [ ] No redundancy with code.
