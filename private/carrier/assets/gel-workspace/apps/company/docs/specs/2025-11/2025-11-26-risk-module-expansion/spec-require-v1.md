# 需求分析：司法风险与经营风险模块扩展

> 所属任务：[司法风险与经营风险模块扩展](./README.md)  
> 文档版本：v1  
> 创建日期：2025-11-11

## 业务背景

当前企业详情页的司法风险和经营风险模块数据维度不够全面，需要补充更多风险指标，帮助用户更全面地评估企业风险状况。同时，为了引流至风控产品，新增的模块采用"统计数字 + 蒙版引流"的方式展示。

### 3. 蒙版引流交互

**展示规则**：

- 仅展示统计数字的目录，不展示列表数据
- 点击后显示蒙版提示，引导用户跳转至风控产品

**跳转要求**：

- 必须直接展示对应企业的对应数据模块所在页面
- 不能只跳到风控的首页
- 需要携带企业标识和模块标识参数

**蒙版文案**（待产品确认）：

```
该数据为风控专属数据
查看完整信息请前往风控产品
[前往查看] [取消]
```

## 功能边界

### 包含范围

- 新增10个风险模块的配置
- 调整司法风险和经营风险模块的顺序
- 实现蒙版引流交互
- 菜单显示统计数字

### 不包含范围

- 不涉及后端接口开发（统计数字由现有接口提供）
- 不涉及风控产品页面开发
- 不涉及现有模块的功能修改

## 验收标准

### 功能验收

- [ ] 司法风险模块新增3个目录，顺序正确
- [ ] 经营风险模块新增7个目录，顺序正确
- [ ] 新增目录仅显示统计数字，不显示列表
- [ ] 点击新增目录显示蒙版提示
- [ ] 点击"前往查看"跳转至风控对应页面
- [ ] 跳转链接携带正确的企业和模块参数
- [ ] 左侧菜单显示统计数字

### 兼容性验收

- [ ] 不影响现有模块的展示和交互
- [ ] 不同企业类型下模块显示正确
- [ ] 海外企业、特殊企业过滤规则正常

### 性能验收

- [ ] 页面加载时间无明显增加
- [ ] 菜单渲染性能无明显下降

## 依赖与约束

### 数据依赖

- 统计数字字段需要后端接口支持（待确认字段名）
- 风控产品跳转链接规则需要产品提供

### 技术约束

- 遵循现有模块配置规范
- 保持与现有代码风格一致
- 不破坏现有功能

### 时间约束

- 需要在下一个迭代完成开发
- 需要充分的测试时间

## 风险识别

| 风险项                   | 影响 | 应对措施                     |
| ------------------------ | ---- | ---------------------------- |
| 统计字段名称未确认       | 高   | 尽快与后端确认字段名         |
| 风控跳转链接规则未明确   | 高   | 尽快与产品确认跳转规则       |
| 模块顺序调整影响现有功能 | 中   | 充分测试，确保不影响现有模块 |
| 蒙版交互体验不佳         | 低   | 与设计师确认交互细节         |
