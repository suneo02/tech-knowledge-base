# 职业发展规划

（Front-end Developer & 海外发展方向）

## 文档定位

- 用途：制定围绕个人画像的可执行方案，明确时间线与指标
- 范围：阶段目标、能力建设、海外策略、风险与年度行动

## 一、阶段目标与里程碑

### 1.1 短期（12–18 个月）

- 总体目标：在国内提升工程化与英语沟通能力，形成对外可验证的技术影响力，为“读硕或直接出国/远程”打基础。
- 关键产出：
  - 工程化专题项目 1 个（Monorepo + pnpm + CI/CD + 性能分析），含技术说明（中/英文）
  - 公开技术文章 3 篇（至少 1 篇英文），主题围绕构建优化、性能与工程化
  - 开源贡献：合并 PR ≥ 6 个（每季度 ≥ 2 个），聚焦 React/TS 生态或工具链
  - 英文简历 + LinkedIn 完成度 100%，GitHub README 英文版完善
- 能力指标：
  - 英语：雅思 6.5（口语 ≥ 6.0）或 TOEFL ≥ 90；每周口语/听力训练 ≥ 4–6 小时
  - 技术：能独立搭建并维护小型 Monorepo，掌握构建分析（bundle/trace）与常见优化方案

### 1.2 中期（18–36 个月）

- 目标：在国内定位为“前端工程化方向的高级工程师候选”，并实质接触国际化团队（远程或海外面试）。
- 关键产出：
  - 主导或核心参与公司工程化项目 1–2 条（如构建系统升级、CI/CD 流水线完善、前端监控/日志体系）
  - 技术分享 2–3 次（公司内或公开），形成可引用的演讲与幻灯资料（中/英文）
  - 海外求职准备：完成 3–5 场英文技术面试模拟与真实面试；建立目标国家岗位清单与联系人网络
- 路线抉择：
  - 路径 A（读硕）：德国/欧洲英文授课硕士优先，预算控制在 20–30 万人民币/年（含生活费），关注毕业后 18 个月求职签证
  - 路径 B（直接海外/远程）：以新加坡/日本/远程为主，要求英语与作品影响力同步提升；积累到 3–5 年经验后窗口更佳

### 1.3 长期（5 年）

- 目标：进入高度国际化团队并在细分方向形成标签（如“前端工程化”“高性能数据可视化”“文档/报表系统”）。
- 能力画像：主导跨团队工程化项目，具备英语会议与技术文档的自然表达，持续输出开源与技术分享。

## 二、能力建设（年度主题）

### 2.1 技术能力

- 工程化体系：Monorepo、CI/CD、模块化与打包优化、依赖治理、前端监控
- 性能优化：大规模表格/报表渲染、可视化性能、SSR/路由与状态管理优化
- 复杂组件：富文本/PDF 报表编辑器的交互与架构设计

### 2.2 语言与软技能

- 英语沟通：每周 4–6 小时听说训练；半年进行 1 次英文技术分享或直播
- 需求与协作：需求拆分、排期估算、跨角色协作；形成规范的技术文档与评审流程

## 三、海外发展策略

### 3.1 国家与路线选择

- 德国/欧洲（读硕优先）：英文授课项目多，学费低；毕业后求职签证常见为 18 个月，适合资金可控的学历+窗口路径。
- 新加坡（直接工作）：岗位国际化，签证与薪资门槛对年资敏感；更适合积累到 3–5 年经验后冲刺。
- 日本（直接工作）：英语岗位有限，日语要求普遍存在；可在具备工程化优势与基础日语后尝试。
- 英语国家（英国/澳大利亚）：学费与生活成本高、签证政策变化；作为高预算备选。

### 3.2 准备清单（非读硕）

- 英文履历与作品集：GitHub 项目、技术博客、技术演讲视频
- 开源协作：选定 1–2 个国际化项目长期贡献，保持稳定频率（每月 ≥ 1 次）
- 海外岗位信息源：远程平台与技术社区，建立岗位跟踪表与联系人列表

### 3.3 读硕路径（备选）

- 预算：控制总花费在 20–30 万人民币/年（含生活费），优先选择英文授课、求职窗口明确的院校与项目
- 关键节点：
  - 语言：雅思/托福达标（建议雅思 ≥ 6.5）
  - 申请材料：英文简历、PS/SOP、推荐信、项目说明（含工程化与性能优化案例）
  - 时间线：预留 6–9 个月准备与申请，入学后积极寻找实习与项目机会

## 四、风险与应对

- 预算风险：设定止损线与资金拆分（学费/生活费/应急金），优先低学费项目与奖学金信息
- 时间窗口风险：并行准备国内竞争力，避免因海外窗口未开启而影响职业成长
- 技术方向风险：每 6 个月进行一次方向复盘（市场需求/个人兴趣/投入产出）
- 求职/签证不确定性：保持多国家多渠道策略（读硕/远程/内部转岗），降低单点失败概率

## 五、年度行动计划（2026）

- Q1：完成 Monorepo 工程化模板与文档；精读 JS/TS 与浏览器原理；完善英文简历与 LinkedIn
- Q2：性能优化专题实践并输出 1 篇英文技术文章；完成开源 PR ≥ 2；进行 1 次公司内技术分享
- Q3：英语听说集中提升（每周 ≥ 6 小时）；完成 GitHub README 英文版；进行 1 次英文技术演讲或录制视频
- Q4：形成《海外发展方案对比表》（德/新/日/英语国家），小规模投递与试面；复盘全年技术与英语指标

## 六、评估方式（OKR 示例）

- O：成为“工程化方向的可验证候选人”
  - KR1：开源 PR 合并 ≥ 6，专题项目 1 个上线并被引用
  - KR2：技术文章 3 篇，其中英文 ≥ 1 篇；公司内技术分享 ≥ 2 次
  - KR3：雅思 ≥ 6.5（口语 ≥ 6.0），完成 3–5 次英文技术面试
